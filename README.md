# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

#### Интерфейс IProduct
Необходим для учета товаров, используемых в приложении.
Содержит данные о товаре, а также типы этих данных: 
`id: string` - ID товара;
`title: string` - название товара;
`description: string` - полное описание товара;
`image: string` - название изображения товара;
`category: string` - категория, к которой относится товар;
`price: number | null` - стоимость товара.

#### Интерфейс IBuyer
Необходим для учета покупателей, заказывающих товар.
Содержит данные о покупателе, а также типы этих днных: 
`payment: 'card' | 'cash' | ''` - выбранный способ оплаты заказа;
`address: string` - адрес доставки заказа;
`e-mail: string` - адрес электронной почты покупателя;
`phone: string` - телефон покупателя.

### Модели данных

#### Класс ProductsList
Отвечает за хранение данных о товарах, которые можно купить в приложении.

Конструктор:
`constructor()` - конструктор не принимает данных.

Поля класса:
`products: IProduct[]` - массив всех товаров, доступных в приложении.
`selectedProduct: IProduct || null` - карточка товара, выбранного для подробного отображения.

Методы класса:
`setProducts(products: IProduct[]): void` - метод для сохранения массива товаров, полученного в параметрах метода.
`getProducts(): IProduct[]` - метод не принимает параметров, возвращает массив товаров из модели.
`setProduct(product: IProduct): void` - метод принимает выбранную карточку и сохраняет ее для подробного отображения.
`getSelectedProduct(): IProduct | null` - метод, для получения выбранного товара. Возвращает null, если ни один из товаров не выбран.
`getProduct(id: string): IProduct | null` - метод принимает id товара и возвращает карточку выбранного товара, возвращает null, если товар не найден.

#### Класс Basket
Отвечает за хранение данных о товарах, которые пользователь выбрал для покупки.

Конструктор:
`constructor()` - конструктор не принимает данных.

Поля класса:
`selectedProducts: IProduct[]` - массив товаров, выбранных покупателем для покупки.

Методы класса:
`addProduct(product: IProduct): void` - метод добавляет товар, полученный в параметре, в корзину.
`removeProduct(id: string): void` - метод удаляет из корзины товар, полученный в параметре.
`getSelectedProducts(): IProduct[]` - метод возвращает список товаров, добавленных в корзину.
`getProductCounter(): number` - метод возвращает количество товаров в корзине.
`getTotalCost(): number` - метод возвращает общую стоимость товаров в корзине. 
`checkProductAvailable(id: string): boolean` - метод для проверки наличия товара в корзине по его id, полученном в параметре.
`clearBasket(): void` - метод для удаления всех товаров из корзины после успешного оформления заказа.

#### Класс Buyer
Отвечает за хранение данных о покупателе, которые тот должен указать при оформлении заказа.

Конструктор:
`constructor()` - конструктор не принимает данных.

Поля класса:
`payment: 'card' | 'cash' | ''` - выбранный способ оплаты заказа.
`address: string` - поле для хранения адреса доставки заказа.
`email: string` - поле для хранения адреса электронной почты покупателя.
`phone: string` - поле для хранения телефона покупателя.

Методы класса:
`getDataOfBuyer(): IBuyer` - метод для получения данных о покупателе.
`setDataOfBuyer(buyer: Partial<IBuyer>): void` - метод для сохранения данных о покупателе.
`removeFormData(): void` - метод для очистки формы от данных покупателя.
`validationDataOfBuyer(): { errors: TBuyerErrors }` - метод для проверки корректности заполненяи каждого поля формы с данными покупателя. В случае невалидного значения метод возвращает объект с указанием конкретного поля с ошибкой и ее описанием.

#### Класс СommunicatorApi
Отвечает за получение данных с сервера и отправку данных на сервер.

Конструктор:
`constructor(api: IApi)` - в конструктор принимается любой объект, реализующий интерфейс IApi.

Поля класса:
`api: IApi` - экземпляр класса, реализующего запросы к серверу.

Методы класса:

### Cобытия
Описание событий, генерирующихся в приложении.

#### События в Моделях данных
`products:change`- событие изменения списка товаров. 
`product:selected` - событие изменения выбранного товара для просмотра.
`basket:changed` - событие изменения содержимого корзины.
`buyer:changed` - событие изменения данных покупателя.
`basket:cleared` - событие очистки корзины от товаров.
`buyer:cleared` - событие очистки данных покупателя.


#### События в Представлениях
`card:select` - событие выбора карточки товара для просмотра (клик по карточке товара в каталоге).
`product:add-to-basket` - событие добавления товара в корзину (клик по кнопке "Купить").
`product:remove-from-basket` - событие добавления товара в корзину (клик по кнопке "Купить").
`basket:open` - событие открытия корзины (клик по иконке корзины в шапке сайта).
`order:open` - событие перехода к окну оформления заказа (клик по кнопке "Оформить" в корзине).
`payment:by-card` - событие выбора способа оплаты по карте (клик по кнопке способа оплаты "Онлайн").
`payment:in-cash` - событие выбора способа оплаты наличными (клик по кнопке способа оплаты "При получении").
`order:address-change` - событие изменения адреса покупателя (ввод текста в поле инпута с адресом доставки).
`order:submit` - событие отправки формы с выбором способа оплаты и адресом покупателя (клик по кнопке "Далее" в первой форме оформления заказа).
`contacts:email-change` - событие изменения электронной почты покупателя (ввод текста в поле инпута с адресом электронной почты).
`contacts:phone-change` - событие изменения телефона покупателя (ввод текста в поле инпута с телефоном).
`contacts:submit` - событие отправки формы с адресом электнной почты и телефоном покупателя (клик по кнопке "Оплатить").
`success:close` - событие закрытия окна с сообщением об успешном оформлении заказа (клик на кнопку "За новыми покупками!").
`modal:open` - событие открытия модального окна.
`modal:close` - событие закрытия модального окна (клик по кнопке закрытия модального окна или клик на оверлей).
